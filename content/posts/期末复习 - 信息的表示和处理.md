## 二、信息存储（§2.1）

### 2.1 比特、字节与字

| 单位 | 英文 | 大小 | 说明 |
|:----:|------|:----:|------|
| 位 | bit | 1 个 0/1 | 信息的最小单位 |
| 字节 | byte | 8 个 bit | C 语言中 `char` 的大小；内存按字节编址 |
| 字 | word | 取决于机器字长 | 32 位机 = 4 字节；64 位机 = 8 字节 |

- 十六进制是二进制最常用的简写：1 个十六进制数字 = 4 个 bit，2 个十六进制数字 = 1 个 byte。例如 `3B` 和 `6D` 是两个 byte，共 16 个 bit。
- 计算机文件分两类：**文本文件**（以 ASCII 标准表示，如 `hello.c`）和**二进制文件**（如 `hello.o`）。

### 2.2 数据类型的大小

C 语言中，不同数据类型在不同平台上的大小可能不同：

| C 类型 | 32 位机器 | 64 位机器 | 备注 |
|:------|:--------:|:--------:|------|
| `char` | 1 | 1 | 总是 1 字节 |
| `short` | 2 | 2 | |
| `int` | 4 | 4 | |
| `long` | 4 | **8** | 32 位和 64 位不同 |
| `long long` | 8 | 8 | |
| `float` | 4 | 4 | |
| `double` | 8 | 8 | |
| `指针 pointer` | **4** | **8** | 指针大小 = 机器字长 |
| `size_t` | 4 | 8 | 无符号整数类型，由平台决定 |

> **关键记忆点**：指针的大小就是机器字长（32 位机 4 字节，64 位机 8 字节）。C99 标准引入了 `int32_t`、`int64_t` 等可移植定长类型，不受平台影响。

### 2.3 字节序：大端 vs 小端

**字节序（Byte Ordering）** 决定了多字节数据在内存中的存放顺序——到底低地址放高位字节还是低位字节？

#### 两种字节序

假设 32 位变量 `x = 0x01234567`，在地址 `0x100` 处存储：

```
大端（Big Endian）：
地址:  0x100    0x101    0x102    0x103
内容:   01       23       45       67
       ↑                           ↑
    最高有效字节                 最低有效字节
    (MSB)                       (LSB)
    低地址放高位字节              高地址放低位字节

小端（Little Endian）：
地址:  0x100    0x101    0x102    0x103
内容:   67       45       23       01
       ↑                           ↑
    最低有效字节                 最高有效字节
    (LSB)                       (MSB)
    低地址放低位字节              高地址放高位字节
```

**记忆口诀**：大端 "人的阅读顺序"（高位在前），小端 "反过来"（低位在前）。x86 和 x86-64 都是**小端**。网络字节序是大端。

#### 为什么需要关心字节序？

1. 跨平台二进制数据传输（网络编程）
2. 阅读机器码/内存 dump 时需要知道大小端
3. 强制类型转换时（如 `int*` 转 `char*`），不同平台会得到不同结果

**典型考题示例**：在小端机器上，`int x = 0x01234567`，用 `char *p = (char *)&x` 读取 `p[0]`，得到 `0x67`（最低字节）。

### 2.4 布尔代数与位运算

C 语言支持的位级运算（按位操作）：

| 运算符 | 含义 | 真值表要点 |
|:------:|------|-----------|
| `&` | 按位与 AND | `1&1=1`，其余为 `0` |
| `|` | 按位或 OR | `0|0=0`，其余为 `1` |
| `~` | 按位取反 NOT | `~1=0`，`~0=1` |
| `^` | 按位异或 XOR | `1^1=0`，`0^0=0`，`0^1=1^0=1` |

**常见用法**：

```c
x & 0xFF          // 取 x 的低 8 位（掩码操作）
x | (1 << n)      // 将 x 的第 n 位置 1
x & ~(1 << n)     // 将 x 的第 n 位置 0
x ^ x             // 结果为 0
a ^ b ^ b         // 结果为 a（对称加密/swap 技巧）
```

#### 逻辑运算 vs 位运算

| 对比 | 逻辑运算 `&&` `||` `!` | 位运算 `&` `|` `~` `^` |
|:---:|:-------------------:|:------------------:|
| 操作对象 | 将非零视为 `true`，零视为 `false` | 对每一位独立操作 |
| 返回结果 | 只返回 `0` 或 `1` | 返回整数（多位结果）|
| 短路求值 | **是**（`&&` 左边为假则不执行右边）| **否** |

```c
// 关键区别示例
0x66 && 0x39   // → 1 (0x01) —— 两个都非零，逻辑与返回 true
0x66 & 0x39    // → 0x20 —— 逐位与：01100110 & 00111001 = 00100000
```

### 2.5 移位运算

| 类型 | 运算符 | 行为 | 图示 |
|:---:|:-----:|------|------|
| 左移 | `x << k` | 全部位左移 k 位，**低位补 0**，高位丢弃 | `0110 << 1 = 1100` |
| 逻辑右移 | `x >> k`（无符号数）| 全部位右移 k 位，**高位补 0** | `1100 >> 1 = 0110` |
| 算术右移 | `x >> k`（有符号数）| 全部位右移 k 位，**高位补符号位（最高位）** | `1100 >> 1 = 1110`（补码下 -4>>1 = -2）|

```c
// 右移类型取决于数据类型，而非操作数的值
unsigned ux = 0x80000000;   // 逻辑右移，高位补 0
  int  sx = 0x80000000;     // 算术右移，高位补 1（符号位）
```

> **性能关键**：移位只需要约 1 个时钟周期，而除法需要约 30 个时钟周期。这就是为什么编译器会把 `x/2` 优化为 `x>>1`。

## 三、整数表示（§2.2）

### 3.1 两种整数编码

假设一个字长为 $w$ 位，位向量 $\vec{x} = [x_{w-1}, x_{w-2}, ..., x_0]$：

#### 无符号编码 B2U（Binary to Unsigned）

$$B2U_w(\vec{x}) = \sum_{i=0}^{w-1} x_i \cdot 2^i$$

最高位 $x_{w-1}$ 的权重为 $+2^{w-1}$。所有位均为非负贡献。

- 取值范围：$[0, 2^w - 1]$
- 对 $w=4$：最小值 `0000` = 0，最大值 `1111` = 15

#### 补码编码 B2T（Binary to Two's Complement）

$$B2T_w(\vec{x}) = -x_{w-1} \cdot 2^{w-1} + \sum_{i=0}^{w-2} x_i \cdot 2^i$$

最高位 $x_{w-1}$ 的权重为 **$-2^{w-1}$**（即符号位），其余位仍为非负贡献。

- 取值范围：$[-2^{w-1}, 2^{w-1} - 1]$
- 对 $w=4$：最小值 `1000` = -8，最大值 `0111` = 7

| w=4 位向量 | B2U 值 | B2T 值 |
|:---------:|:------:|:------:|
| `0000` | 0 | 0 |
| `0001` | 1 | 1 |
| `0111` | 7 | 7 |
| `1000` | 8 | **-8** |
| `1001` | 9 | **-7** |
| `1111` | 15 | **-1** |

**关键观察**：补码的负数区域中，`100...00` 是最小的负数（$2^{w-1}$），`111...11` 是 -1。

#### 关键特殊值

| 值 | 位模式 | 性质 |
|:--:|:------:|------|
| `INT_MAX` | `011...11` | 最大有符号正数 = $2^{w-1}-1$ |
| `INT_MIN` | `100...00` | 最小有符号负数 = $-2^{w-1}$ |
| `-1` | `111...11` | 补码下全 1 = -1 |
| `0` | `000...00` | 零的补码只有一种表示 |
| `UMAX` | `111...11` | 无符号最大值 = $2^w-1$ |

> **补码的不对称性**：`|INT_MIN| = INT_MAX + 1`。即负数范围比正数多一个（负方向多一个数）。

### 3.2 取反（Negation）

不管是有符号还是无符号整数，**取反（变负）操作都是将所有位（包括最高位）取反再加 1**：

```c
// 两个重要的边界情况：

// 1. 对 0 取反：~0 + 1 = 111...11 + 1 = 000...00 = 0 ✓
//    （进位导致各位归零）

// 2. 对 INT_MIN 取反：~INT_MIN + 1 = 011...11 + 1 = 100...00 = INT_MIN
//    （溢出导致原值不变！这是常见的程序 bug）
int b;
scanf("%d", &b);
if (b <= 0)
    printf("%d\n", -b);   // 输入 INT_MIN 时，输出仍然是 INT_MIN！
```

> **INT_MIN 取反不变是经典陷阱**——`abs(INT_MIN)` 返回的是 `INT_MIN` 本身。

### 3.3 有符号与无符号之间的转换

**核心原则**：类型转换**不改变位级表示**，只改变对这些位的**解释方式**。即"Keep bit representations and reinterpret."
$$
T2U_w(x) = \begin{cases}
x & \text{if } x \geq 0 \\
x + 2^w & \text{if } x < 0
\end{cases}
\qquad
U2T_w(u) = \begin{cases}
u & \text{if } u \leq 2^{w-1}-1 \\
u - 2^w & \text{if } u \geq 2^{w-1}
\end{cases}
$$

```c
// 关键示例
short sx = -12345;       // 补码：0xCFC7
unsigned uy = sx;        /* 转换步骤（教科书级）：
                            1. 先符号扩展：sx 是 short（16位），赋值给 int 要先扩展到 32 位
                               扩展后：0xFFFFCFC7（符号位复制）
                            2. 再 reinterpret 为 unsigned：得到 4294954951
                            不是：先转 unsigned 再补 0！这是常见错误理解 */

unsigned ux = 4294954951;
int      x  = (int) ux;  // reinterpret 为有符号 → -12345
```

#### 隐式转换的陷阱

**当表达式中同时出现有符号和无符号数时，C 语言会隐式将有符号数转换为无符号数。** 这是非常容易出 bug 的地方。

```c
// 陷阱 1：无符号循环变量
unsigned i;
for (i = n - 1; i >= 0; --i)   // 无限循环！i>=0 恒为真（无符号永不小于0）
    ...

// 陷阱 2：sizeof 返回无符号数
int j;
for (j = n - 1; j - sizeof(char) >= 0; --j)   // 无限循环！
    // sizeof 返回 size_t（无符号），整个表达式被隐式转为无符号

// 正确写法
size_t i;  // 或 unsigned int i
for (i = n - 1; i < n; --i)    // 当 i 减到 0 再减 1 时，无符号回绕到 UMAX
    ...                         // 此时 i < n 为假，循环终止
```

**判断原则**：只要表达式中有一个无符号数，整个表达式就按无符号规则计算——比较、加减等全部当作无符号进行。

### 3.4 位扩展：$w$ 位 → $w+k$ 位

两种扩展方法：

| 方法 | 适用场景 | 操作 |
|:---:|---------|------|
| **零扩展** | 无符号数 | 高位直接补 k 个 0 |
| **符号扩展** | 补码有符号数 | 高位复制符号位（$x_{w-1}$）k 次 |

**符号扩展为什么正确？** 数学验证（补码下）：

$$B2T_{w+1}([x_{w-1}, x_{w-1}, x_{w-2}, ..., x_0]) = -x_{w-1} \cdot 2^w + x_{w-1} \cdot 2^{w-1} + \sum_{i=0}^{w-2} x_i \cdot 2^i$$
$$= -x_{w-1} \cdot (2^w - 2^{w-1}) + \sum_{i=0}^{w-2} x_i \cdot 2^i$$
$$= -x_{w-1} \cdot 2^{w-1} + \sum_{i=0}^{w-2} x_i \cdot 2^i = B2T_w(\vec{x})$$

```c
// 经典考题：类型转换的顺序
short sx = -12345;          // 16位：0xCFC7
unsigned uy = sx;           /* 步骤（极易出错！）：
                                1. short → int：符号扩展 → 32位 0xFFFFCFC7
                                2. int  → unsigned：reinterpret → 4294954951 */
```

> **实用判断**：如果你发现一个十六进制数的前几位出现了连续的 `F`，那么这个数在补码下一定是负数。

### 3.5 位截断：$w+k$ 位 → $w$ 位

直接去掉最高 k 位，保留低 w 位。

对无符号数：$\text{trunc}(x) = x \bmod 2^w$
对补码：$\text{trunc}(x) = U2T_w(x \bmod 2^w)$

```c
int   x = 53191;        // 0x0000CFC7
short y = (short) x;    // 截断为 0xCFC7 = -12345 (补码)
// 截断可能导致：正数变负数、数值完全改变
```

## 四、整数运算（§2.3）

### 4.1 核心原则：模运算截断

不管是无符号还是有符号整数，**溢出时统一处理为"将超出 w 位的部分直接舍去"**。在 C 语言中，无符号整数溢出是**已定义的**行为（模 $2^w$），而有符号整数溢出属于**未定义行为（undefined behavior）**——虽然实践中也常表现为截断。

### 4.2 无符号加法

$$x +^u_w y = (x + y) \bmod 2^w$$

正常结果：$x + y < 2^w$ → 不溢出
溢出：$x + y \geq 2^w$ → 结果 = $x + y - 2^w$（截断高位）

**溢出检测**：

```c
unsigned a, b, ans;
ans = a + b;
if (ans < a || ans < b)   // 如果和小于任一加数，说明发生了溢出
    printf("Overflow.\n");
```

### 4.3 补码加法

$$x +^t_w y = U2T_w((B2T_w(x) + B2T_w(y)) \bmod 2^w)$$

补码的加法仍按无符号的规则做二进制加法并截断，只是最后 reinterpret 为有符号。

**四种情况判断**：

| x | y | x+y | 是否溢出 | 结果 |
|:--:|:--:|:--:|:------:|------|
| + | + | $\geq 2^{w-1}$ | **正溢出** → 结果变成负数 | $x+y-2^w$ |
| + | + | $< 2^{w-1}$ | 正常 | $x+y$ |
| + | - | — | **不可能溢出** | $x+y$ |
| - | - | $< -2^{w-1}$ | **负溢出** → 结果变成正数 | $x+y+2^w$ |
| - | - | $\geq -2^{w-1}$ | 正常 | $x+y$ |

```c
// 示例：0x7fffffff + 0x7fffffff → 0xFFFFFFFE → -2（正溢出）
// 示例：0x80000000 + 0x80000000 → 0x00000000 →  0（负溢出）
```

**溢出检测**：

```c
int A, B, Ans;
Ans = A + B;
// 正溢出：两正相加得负；负溢出：两负相加得正
if ((A > 0 && B > 0 && Ans <= 0) || (A < 0 && B < 0 && Ans >= 0))
    printf("Overflow.\n");
```

### 4.4 补码取反（再强调）

$$-^t_w x = \begin{cases}
-2^{w-1} & \text{if } x = -2^{w-1} \quad\text{(INT\_MIN 取反仍是自身！)} \\
\text{~}x + 1 & \text{otherwise}
\end{cases}$$

### 4.5 无符号与补码乘法

乘法结果的位数是两操作数位数之和（$2w$ 位），但 C 语言只保留低 $w$ 位（截断）：

$$x *^u_w y = (x \cdot y) \bmod 2^w$$
$$x *^t_w y = U2T_w((x \cdot y) \bmod 2^w)$$

**关键推论**：补码乘法和无符号乘法在**位级是完全相同的**——同样的二进制运算 + 同样的截断，只是最后 interpret 不同。

### 4.6 乘以常数 → 移位+加减

编译器从不使用乘法指令来乘常数，而是优化为移位和加减组合（乘法指令 ~3 个周期，移位 ~1 个周期）：

| 乘法 | 等价移位操作 | 公式 |
|:---:|------------|------|
| `x * 2` | `x << 1` | $x \cdot 2^k$ = `x << k` |
| `x * 3` | `(x << 1) + x` | $x \cdot (2^k + 1) = (x <\!< k) + x$ |
| `x * 5` | `(x << 2) + x` | |
| `x * 9` | `(x << 3) + x` | |

### 4.7 除以 2 的幂

**无符号除法**（逻辑右移，向零舍入）：

$$x / 2^k = x >\!> k \quad\text{(逻辑右移，高位补 0)}$$

**补码除法**（算术右移 + 偏置修正，向零舍入）：

$$x / 2^k = \begin{cases}
x >\!> k & \text{if } x \geq 0 \\
(x + (1 <\!< k) - 1) >\!> k & \text{if } x < 0
\end{cases}$$

> **为什么负数需要加偏置？** 补码的算术右移结果向 **$-\infty$ 舍入**（即向下取整），但 C 语言的整数除法要求**向零舍入**。例如 `-7 / 4` 在数学上是 `-1.75`，向零舍入 = `-1`，但算术右移 `-7 >> 2` 得到 `-2`（向下取整）。通过先加 `(1<<k)-1 = 3` 的偏置：`(-7+3) >> 2 = (-4) >> 2 = -1`，修正了舍入方向。

$$
\begin{aligned}
x = 1001 &= -7 \\
x >> 2 &\rightarrow 1110 = -2 \quad\text{(算术右移，向-∞舍入)} \\
(x + 3) >> 2 &= (-7 + 3) >> 2 = -4 >> 2 = -1 \quad\text{(修正后，向零舍入 ✓)}
\end{aligned}
$$

> **除法 vs 位移的性能差异**：除法需要约 30 个时钟周期，位移只需约 1 个。编译器大量利用这一优化。

## 五、浮点数（§2.4）

### 5.1 IEEE 754 浮点表示

浮点数格式：**$V = (-1)^s \times M \times 2^E$**

| 精度 | 符号 s | 阶码 exp | 尾数 frac | 总位数 | 偏置 Bias |
|:---:|:-----:|:-------:|:--------:|:-----:|:--------:|
| 单精度 float | 1 bit | 8 bits (k=8) | 23 bits (n=23) | 32 bits | $2^7-1 = 127$ |
| 双精度 double | 1 bit | 11 bits (k=11) | 52 bits (n=52) | 64 bits | $2^{10}-1 = 1023$ |

```
单精度格式:
  | s (1) |    exp (8)    |         frac (23)            |
双精度格式:
  | s (1) |     exp (11)     |             frac (52)                |
```

### 5.2 三种情况

根据阶码 exp 的值，分为三类：

| 情况 | exp | frac | $E$（阶）| $M$（尾数）| 说明 |
|:---:|:---:|:----:|:--------:|:---------:|------|
| **规格化** | $\neq 0$ 且 $\neq \text{全}1$ | 任意 | $e - \text{Bias}$ | $1 + f$ | 最常规的浮点数（隐含 1）|
| **非规格化** | 全 0 | 任意 | $1 - \text{Bias}$ | $f$ | 表示 0 及非常接近 0 的数（**无隐含 1**）|
| **特殊值** | 全 1 | 全 0 | — | — | $\pm \infty$（无穷大）|
| **特殊值** | 全 1 | $\neq 0$ | — | — | $\text{NaN}$（Not a Number）|

> **为什么非规格化的 E = 1-Bias 而不是 0-Bias？** 这是 IEEE 754 的精妙设计——保证了从非规格化到规格化的**平滑过渡**。最大的非规格化数到最小的规格化数之间没有"跳变"。

### 5.3 关键数值（以单精度 32-bit 为例）

| 位模式 (hex) | s | exp (e) | frac (f) | 十进制值 | 分类 |
|:-----------|:-:|:-------:|:--------:|:-------:|:---:|
| `0x00000000` | 0 | 0 | 0 | 0.0 | 非规格化 |
| `0x80000000` | 1 | 0 | 0 | -0.0 | 非规格化 |
| `0x3F800000` | 0 | 127 | 0 | 1.0 | 规格化 $(-1)^0 \times 1.0 \times 2^{127-127}$ |
| `0xBF800000` | 1 | 127 | 0 | -1.0 | 规格化 |
| `0x7F800000` | 0 | 255 | 0 | $+\infty$ | 特殊值 |
| `0xFF800000` | 1 | 255 | 0 | $-\infty$ | 特殊值 |
| `0x7F800001` | 0 | 255 | $\neq 0$ | $\text{NaN}$ | 特殊值 |
| `0x00000001` | 0 | 0 | $2^{-23}$ | $1.4\times 10^{-45}$ | 非规格化（最小正值）|
| `0x7F7FFFFF` | 0 | 254 | 全 1 | $\approx 3.4\times 10^{38}$ | 规格化（最大正值）|

### 5.4 舍入（Rounding）

IEEE 754 定义了四种舍入模式，默认是**向偶数舍入**（Round-to-Even，也叫"银行家舍入"）：

- 中间值（如 `1.5`）：舍入到最接近的偶数（`2.0`）
- 非中间值：舍入到最接近的数
- 偶数舍入的优势：统计上无偏，避免累积误差

| 值 | 向偶数舍入到整数 |
|:--:|:--------------:|
| 1.4 | 1 |
| 1.5 | **2**（最近的偶数是 2，不是 1）|
| 1.6 | 2 |
| 2.5 | **2**（最近的偶数是 2，不是 3）|
| -1.5 | **-2**（最近的偶数是 -2）|

### 5.5 浮点运算

浮点运算**不满足结合律**（由于舍入）：

```c
// 这些等式在浮点数下不成立！
(a + b) + c  ≠  a + (b + c)     // 结合律不成立
a * (b + c)  ≠  a*b + a*c       // 分配律不成立
(3.14 + 1e20) - 1e20  →  0.0   // 不是 3.14！（加法中精度丢失）
3.14 + (1e20 - 1e20)   →  3.14  // 正确
```

浮点加法满足**交换律**：$a + b = b + a$。浮点乘法满足**交换律**，但不满足结合律和分配律。

### 5.6 整数与浮点之间的转换

```c
int   x = ...;
float f = ...;
double d = ...;

// int → float：可能发生舍入（int 最多 32 位，float 尾数只有 23 位）
// int → double：只要 int ≤ 53 位（double 尾数 52+1），精确
// float/double → int：向零舍入（截断小数部分），可能溢出
```

| 转换 | 是否安全 | 原因 |
|:---:|:------:|------|
| `int` → `float` | **可能舍入** | float 尾数 23 位 < int 的 32 位 |
| `int` → `double` | **精确** | double 尾数 52+1 位 > int 的 32 位 |
| `float` → `int` | **可能溢出+截断** | `(int) float` 强制截断 |

```c
// 经典坑：(int) (float) x 可能不等于 x
int x = 2147483647;                   // 接近 INT_MAX
(float) x;                            // 舍入为 2.14748365×10^9 (有效位丢失!)
(int) (float) x;                      // 可能 ≠ 原值
```

### 6.1 本章核心原理一句话

> **"数据的位级表示 + 上下文（类型）决定了解释方式"** —— 同一串 bit 可以表示无符号整数、补码整数、浮点数、指令、字符串...取决于你怎么看它。
