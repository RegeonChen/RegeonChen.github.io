## 二、存储技术（§6.1–§6.2）

### 2.1 三种存储技术的比较

| 技术 | 典型访问时间 | 成本 | 是否易失 | 典型用途 |
|:---:|:---------:|:---:|:------:|---------|
| **SRAM** | ~4–10 ns | 高（每 bit 6 个晶体管）| 易失 | L1/L2/L3 缓存 |
| **DRAM** | ~50–100 ns | 中（每 bit 1 个晶体管+1 个电容）| 易失 | 主存（内存条）|
| **磁盘/SSD** | ~5–10 ms | 低 | 非易失 | 持久化存储、swap space |

> **关键数量级差距**：磁盘比 DRAM 慢约 **100,000 倍**，DRAM 比 SRAM 慢约 **10–20 倍**。正是这种巨大的速度差催生了存储器层次结构。

### 2.2 SRAM（Static RAM）

- 使用**双稳态触发器**存储 bit，只要有电就保持状态
- 访问速度快，和 CPU 在同一芯片或紧邻封装
- 每 bit 占用 6 个晶体管 → 密度低、成本高 → 只能做小容量缓存
- L1 缓存典型大小：32 KB；L2：256 KB；L3：8–32 MB

### 2.3 DRAM（Dynamic RAM）

DRAM 使用**电容**存储 bit，电容会漏电 → 必须周期性刷新（refresh）。每 bit 只需 1 个晶体管 + 1 个电容 → 密度高、成本低，典型主存大小 8–64 GB。

#### 2.3.1 DRAM 芯片的内部组织

单个 DRAM 芯片内部是一个**矩形二维阵列**，由 **d 个超单元（supercell）** 组成，每个超单元存储 **w 位**：

```
参数定义：
  r  = 行数 (rows)
  c  = 列数 (columns)
  d  = r × c    （总超单元数）
  w  = 每个超单元的位数（通常 4 或 8 位）

芯片总容量 = d × w bits = (r × c) × w bits
```

```
DRAM 芯片内部结构（以 d=16, r=4, c=4, w=8 为例）：

           列 0      列 1      列 2      列 3
         ┌────────┬────────┬────────┬────────┐
   行 0  │ (0,0)  │ (0,1)  │ (0,2)  │ (0,3)  │
         │  8bits │        │        │        │
         ├────────┼────────┼────────┼────────┤
   行 1  │ (1,0)  │ (1,1)  │ (1,2)  │ (1,3)  │
         │        │        │        │        │
         ├────────┼────────┼────────┼────────┤
   行 2  │ (2,0)  │ (2,1)  │ (2,2)  │ (2,3)  │
         │        │        │        │        │
         ├────────┼────────┼────────┼────────┤
   行 3  │ (3,0)  │ (3,1)  │ (3,2)  │ (3,3)  │
         │        │        │        │        │
         └────────┴────────┴────────┴────────┘
                          ↑
              每个 (i,j) 是一个 supercell，
              存储 w=8 位数据
```

- **超单元 (i, j)**：位于第 i 行、第 j 列的存储单元，它存储 w 位（w 是芯片的数据引脚数）
- 芯片通过 **addr 引脚** 接收行列地址，通过 **data 引脚** 读写 w 位数据

#### 2.3.2 两阶段地址发送

由于 DRAM 阵列很大，行列地址**复用同一组地址引脚**，分两次发送：

```
  RAS (Row Access Strobe)：
    → 发送行地址 i → 芯片将第 i 行整行的所有 c 个 supercell
      复制到内部行缓冲区（row buffer）

  CAS (Column Access Strobe)：
    → 发送列地址 j → 从行缓冲区中选出第 j 列的 supercell
    → 通过 data 引脚读出或写入 w 位
```

```
时序示意：

  时钟: ─┬─────┬─────┬─────┬─────┬─────┬─────
        │ RAS │     │ CAS │     │     │
  地址线: │ Row │     │ Col │     │     │
        │Addr i│     │Addr j│     │     │
  数据线: │     │ ... │     │ w bits read/written │
         ─┴─────┴─────┴─────┴─────┴─────┴─────
                   ↑              ↑
              RAS 延迟        CAS 延迟
```

**为什么按行缓冲？** 整行被读入行缓冲区后，同一行内不同列的连续访问（如 CAS 反复选中不同列）不需要再次发送 RAS——这利用了空间局部性，称为**快速页模式（Fast Page Mode）**。

#### 2.3.3 内存模块：多片 DRAM 芯片组合成 DIMM

单个 DRAM 芯片的数据宽度 w 很小（4 或 8 位），而 CPU 的数据总线是 64 位。解决办法：**多片 DRAM 芯片并联**组成一个内存模块（DIMM）。

```
例：用 8 片 w=8 的 DRAM 芯片组成一个 64 位宽的 DIMM：

  CPU 发出 64 位地址 A
       │
       ▼
  内存控制器（Memory Controller）
       │  将地址 A 拆分为 RAS/CAS 信号
       │  同时发送给所有 8 片芯片
       │
       ▼
  ┌──────┐ ┌──────┐ ┌──────┐        ┌──────┐
  │Chip 0│ │Chip 1│ │Chip 2│  ...   │Chip 7│
  │ w=8  │ │ w=8  │ │ w=8  │        │ w=8  │
  └──┬───┘ └──┬───┘ └──┬───┘        └──┬───┘
     │        │        │               │
     └────────┴────────┴───────────────┘
                      │
              64 位数据总线 → CPU
```

所有芯片**共享同一组地址和控制信号**（RAS/CAS/片选），但各自提供 8 位数据。8 片 × 8 位 = 64 位，一个地址同时读出 64 位数据。

在这种架构下，地址 A 的 **supercell (i, j)** 在芯片 k 上存储的是完整数据字的第 `[k*8, k*8+7]` 位。

#### 2.3.4 从物理地址到 DRAM 行列的映射

内存控制器接收物理地址后，将其拆分为：

```
物理地址 = | Rank | Bank | Row | Column |
```

- **Rank**：选择 DIMM 上的哪一组芯片（一个 DIMM 可以有 1 或 2 个 rank）
- **Bank**：选择芯片内部的哪个存储体（bank 之间可以并行，隐藏延迟）
- **Row / Column**：同 §2.3.2，分别送入 RAS 和 CAS

> **历史注脚**：传统 DRAM（如 FPM、EDO）是异步的——CPU 需要插入等待周期来等待 DRAM 完成操作。现代 **SDRAM（Synchronous DRAM）** 和 **DDR SDRAM** 使用时钟信号同步，在时钟的上升沿和/或下降沿传输数据，大幅提高了带宽。DDR = Double Data Rate（双倍数据速率），每个时钟周期传输两次数据。

### 2.4 磁盘存储

#### 机械硬盘（HDD）

##### 物理组成与读数据的过程

磁盘由**盘片**（platter）、**磁道**（track）、**扇区**（sector）组成。盘片高速旋转，磁臂驱动磁头在盘片上方径向移动来读写数据。

读取一个目标扇区的完整物理过程分为四步：

1. **寻道（Seek）**：磁臂驱动磁头进行径向移动，从当前磁道移动到目标扇区所在的磁道上方。这是纯机械运动。
2. **旋转延迟（Rotational Latency）**：磁头已在正确磁道上，但目标扇区还没转到磁头下方——磁头必须等待盘片旋转，直到目标扇区转到正下方。
3. **数据传输（Data Transfer）**：目标扇区转到磁头下方，磁头开始感应磁场，读取扇区中的所有位。
4. 如果还要读下一个扇区（不在同一磁道上），重复步骤 1–3。

##### 总访问时间公式

磁盘访问一个目标扇区的平均总时间由三部分组成：

$$T_{access} = T_{avg\ seek} + T_{avg\ rotation} + T_{avg\ transfer}$$

| 组成部分 | 定义 | 计算公式 / 典型值 | 备注 |
|:---|------|:--------------|------|
| **寻道时间** | 磁头移动到目标磁道的时间 | **3–9 ms**（典型值）| 机械运动，受限于磁臂物理速度 |
| **旋转延迟** | 等待目标扇区转到磁头下方 | $T_{avg\ rotation} = \frac{1}{2} \times \frac{1}{\text{RPM}} \times 60$ 秒 | 假设随机分布，平均转半圈；7200 RPM 下约 **4 ms** |
| **传输时间** | 读写扇区中所有位的时间 | $T_{avg\ transfer} = \frac{1}{\text{RPM}} \times \frac{1}{\text{每磁道平均扇区数}} \times 60$ 秒 | 非常短，约 **0.02 ms** |

##### 关键结论

- **瓶颈所在**：磁盘访问时间主要被**寻道时间**和**旋转延迟**占据——两者都是毫秒级。传输时间相比之下极短（微秒级）。
- **"第一比特最贵"**：获取扇区的第一个 bit 成本最高（需要经历寻道和旋转等待），一旦磁头就位，读取剩余数据几乎是"免费"的（非常快）。
- **巨大的速度差距**：
  - 磁盘比 SRAM 慢约 **40,000 倍**（SRAM 访问 ~4 ns，磁盘 ~10 ms）
  - 磁盘比 DRAM 慢约 **2,500 倍**（DRAM 访问 ~60 ns，磁盘 ~10 ms）
- 顺序访问快（同一磁道无需寻道），随机访问极慢（每次都要寻道+旋转等待）

#### 系统级 I/O 交互：CPU 如何通过总线读取磁盘数据

磁盘本身很慢，但计算机系统通过**总线架构**和**DMA机制**让 CPU 不被拖累——仅在发起和完成时介入，中间全由硬件自动搬运数据。

##### 硬件互连架构

现代计算机的 I/O 系统由以下组件通过总线连接：

```
  ┌─────────┐
  │ CPU 芯片 │
  └────┬────┘
       │ 系统总线
       ▼
  ┌──────────┐
  │ I/O 桥接器 │ （Intel 平台上是"北桥"或集成到 CPU 内）
  └──┬───┬───┘
     │   │
     │   └──────────────┐
     ▼ 内存总线          ▼ I/O 总线（PCIe / SATA / USB）
  ┌──────┐    ┌──────┬──────┬──────┐
  │ 主存  │    │磁盘  │ USB  │ 显卡  │
  │ DRAM │    │控制器│控制器│控制器│
  └──────┘    │+磁盘 │      │      │
              └──────┘      └──────┘
```

- **I/O 桥接器**连接 CPU、主存和 I/O 设备，负责总线协议转换和数据转发
- **磁盘控制器**是磁盘与 I/O 总线的接口，内部有寄存器和缓冲区来接收命令、参与 DMA

##### 读取磁盘的三个阶段（"命令—DMA—中断"模型）

**阶段 1：CPU 发起命令**

CPU 向磁盘控制器写入：① 命令（READ）、② 逻辑块号（要读哪些扇区）、③ 目的内存地址（数据放哪儿）。CPU 是主动方，写入后可以继续做别的事。

**阶段 2：DMA 数据传输（磁盘控制器独立工作）**

这是整个过程最关键的设计——**DMA（Direct Memory Access，直接内存访问）**：

- 磁盘控制器**接管 I/O 总线和内存总线**，直接把数据从磁盘搬运到主存——**CPU 完全不参与，可以并行执行其他进程**
- 如果没有 DMA，所有数据都必须先搬运到 CPU 寄存器再写回内存（PIO 模式），CPU 被白白占用

```
  传统 PIO 模式：                   DMA 模式：
  磁盘 → CPU → 内存                磁盘 → 内存
  CPU 全程被占用                  CPU 只在头尾介入
```

**阶段 3：中断通知（控制器 → CPU）**

DMA 传输完成后，磁盘控制器发出**硬件中断**（interrupt）信号通知 CPU。CPU 暂停当前工作，确认数据已就绪（第 8 章 ECF），随后继续处理。这就是第 8 章 §8.1 中断异常的经典实例——异步、来自外部 I/O 设备。

> **一句话总结**：CPU 发起命令（写几个寄存器）→ 磁盘控制器通过 DMA 自己搬数据到主存 → 完成后发中断通知 CPU。CPU 从头到尾只碰了两次磁盘控制器，其余时间完全空闲跑别的进程。

#### 固态硬盘（SSD）

固态硬盘基于 NAND 闪存，无机械部件，消除了寻道和旋转延迟。随机访问比 HDD 快约 100 倍（~0.1 ms vs ~10 ms）。

##### 内部架构

```
  主机 (CPU)
     │  逻辑块地址 (Logical Block Address)
     ▼
  ┌──────────────┐
  │ 闪存翻译层    │ ← FTL (Flash Translation Layer)
  │  (FTL)       │    逻辑→物理映射 + 磨损均衡 + 垃圾回收
  ├──────────────┤
  │ DRAM 缓冲区   │ ← 缓存映射表 + 热点数据
  ├──────────────┤
  │ NAND 闪存芯片  │ ← 实际存储介质
  │  Chip 0 ... n │
  └──────────────┘
```

**FTL（Flash Translation Layer）** 是 SSD 的核心智能组件，位于主机接口和闪存芯片之间，负责：

- **地址映射**：将主机发出的"逻辑块号"翻译为闪存内部的"物理块号"——这是第 9 章虚拟内存的映射思想在硬件层面的再现
- **磨损均衡（Wear Leveling）**：将写入均匀分散到所有闪存块，避免某些块因频繁擦写而提前报废
- **垃圾回收（Garbage Collection）**：回收被标记为无效的旧页

##### 闪存的组织结构与操作约束

闪存芯片内部的两级结构：

```
  闪存芯片
    ├── 块 (Block)：擦除的最小单位
    │     ├── 页 (Page)：读/写的最小单位 (512B – 4KB)
    │     ├── 页
    │     └── ...     (一个块通常有 32–128 个页)
    ├── 块
    └── ...
```

| 操作 | 粒度 | 说明 |
|:---:|:---:|------|
| **读** | 页 | 快速，几十微秒 |
| **写** | 页 | 需要目标页已被擦除 |
| **擦除** | **块**（整块！）| 最慢的操作，几毫秒；擦除后才能重新写入 |

**核心痛点："写入前必须擦除"**：

- 要修改一个页的数据，不能直接覆盖——必须先将整个块的内容读到缓冲区 → 擦除整块 → 把修改后的内容写回
- 如果一个块中有 128 个页，想改其中 1 页，其余 127 页也必须一起"陪葬"（读出→擦除→写回）
- 这就是 FTL 存在的原因：FTL 将新数据写到**已经被预先擦除好的新块**，旧页标记为"无效"，后续由垃圾回收统一回收整块

##### 寿命与磨损均衡

闪存块的擦除次数有限（通常约 10,000–100,000 次）。**磨损均衡**算法确保所有块被均匀使用——当某个页被频繁修改时，FTL 将新版本写到不同的物理块上，而不是反复擦写同一块。这就是"为什么 SSD 芯片容量越用越小"——随着时间推移，越来越多的块达到寿命上限被标记为不可用。

##### HDD vs SSD 总结对比

| 特性 | HDD（机械磁盘）| SSD（固态硬盘）|
|:---:|:------------:|:----------:|
| 随机访问延迟 | ~10 ms（寻道+旋转）| ~0.1 ms |
| 瓶颈来源 | 机械运动 | 擦除操作 |
| 写入方式 | 直接覆盖 | 不可覆盖，需先擦除整块 |
| 寿命限制 | 无明确次数 | 擦除次数限制 → 需磨损均衡 |
| 控制器复杂度 | 低（简单扇区映射）| 高（FTL：映射+磨损均衡+垃圾回收）|

> **故事脉络**：机械磁盘受物理运动限制 → I/O 总线和 DMA 让 CPU 不被拖累 → SSD 用电子存储消除了机械延迟，但引入了新的"擦除约束" → FTL 用类似第 9 章虚拟内存的映射思想，在上层掩盖闪存的物理限制。

## 三、局部性原理（§6.3）

局部性是存储器层次结构有效的**根本原因**——如果程序随机访问内存，缓存就没用了。

### 3.1 时间局部性（Temporal Locality）

**刚被访问过的数据，很可能很快再次被访问。**

```c
// 例：循环中的 sum 变量
int sum = 0;
for (int i = 0; i < n; i++)
    sum += a[i];   // sum 每次迭代都被读写 → 时间局部性
```

- 循环变量、累加器、频繁调用的函数——这些在缓存中热得很
- 缓存正是利用时间局部性：把最近访问的数据留在高速缓存中，再次访问时命中

### 3.2 空间局部性（Spatial Locality）

**刚被访问的数据附近的数据，很可能很快被访问。**

```c
// 例：遍历数组
for (int i = 0; i < n; i++)
    sum += a[i];   // a[0], a[1], a[2]... 地址连续 → 空间局部性
```

- 数组、结构体、连续指令（PC 顺序递增）——都展现空间局部性
- 缓存利用空间局部性：一次加载一整块（cache block = 64 字节），把附近的数据一起带进来

### 3.3 步长访问对局部性的影响

```c
// 良好空间局部性：步长 = 1，顺序访问
int sum1(int a[M][N]) {
    int s = 0;
    for (int i = 0; i < M; i++)
        for (int j = 0; j < N; j++)
            s += a[i][j];   // 行优先遍历 → 步长 1 → 空间局部性极好
    return s;
}

// 较差空间局部性：步长 = N，跨行访问
int sum2(int a[M][N]) {
    int s = 0;
    for (int j = 0; j < N; j++)
        for (int i = 0; i < M; i++)
            s += a[i][j];   // 列优先遍历 → 步长 N → 缓存不命中多
    return s;
}
```

> **行优先 vs 列优先**：C 语言数组按行存储，`a[i][j]` 和 `a[i][j+1]` 地址相邻。按行遍历每一步都在同一个 cache block 内（步长 = 1 = 最佳空间局部性），按列遍历每一步跳到下一行（步长 = N × sizeof(element)，大概率跨 cache block = 大量 miss）。

### 3.4 取指令的局部性

代码本身也展现局部性：
- **空间局部性**：CPU 顺序执行指令，PC 递增 → 连续地址的指令几乎总在同一个 cache block 中（因此有独立的 **L1-I 指令缓存**）
- **时间局部性**：循环体会被反复执行，`for`/`while` 里的指令短期内被多次访问

## 四、存储器层次结构（§6.4）

### 4.1 存储金字塔

```
           ┌────────────┐
           │  寄存器     │  ← CPU 内部，0 周期延迟，~1 KB
           ├────────────┤
           │  L1 缓存   │  ← SRAM，~4 周期，32 KB
           ├────────────┤
           │  L2 缓存   │  ← SRAM，~10 周期，256 KB
           ├────────────┤
           │  L3 缓存   │  ← SRAM，~40 周期，8–32 MB（多核共享）
           ├────────────┤
           │  主存 DRAM │  ← ~200 周期，8–64 GB
           ├────────────┤
           │  本地磁盘  │  ← ~10 ms（= 10,000,000 周期），~TB
           ├────────────┤
           │  远程存储  │  ← 网络，更高延迟
           └────────────┘
           速度 ↑                   容量 ↑
           价格 ↑                   价格 ↓
```

### 4.2 缓存的基本原理

每一层都是下一层的**缓存**（cache）：

- **L1** 是 L2 的缓存
- **L2** 是 L3 的缓存
- **L3** 是主存（DRAM）的缓存
- **主存**是磁盘的缓存（第 9 章虚拟内存的核心思想）

**命中（Hit）**：所需数据刚好在较快的存储层中 → 直接从该层读取
**不命中（Miss）**：数据不在 → 到下一层去找，找到后**拷贝一个块**到当前层（可能驱逐旧块）

### 4.3 缓存不命中的三种类型（3C 模型）

| 类型 | 英文 | 发生原因 | 缓解方法 |
|:---:|------|---------|---------|
| **冷不命中** | Cold (Compulsory) Miss | **第一次**访问某个块，缓存为空 | 无法避免；可通过预取（prefetch）缓解 |
| **容量不命中** | Capacity Miss | 缓存**太小**，装不下工作集 | 增大缓存 |
| **冲突不命中** | Conflict Miss | 多个块映射到**同一组**，互相挤占 | 增加相联度（E），或用全相联 |

## 五、缓存组织结构（§6.5）

### 5.1 缓存的四个关键参数

| 参数 | 含义 | 典型值 |
|:---:|------|:-----:|
| **S** = $2^s$ | 组的数量 | 64–8192 |
| **E** | 每组的行数（相联度）| 1（直接映射）/ 8（8 路组相联）/ 全相联 |
| **B** = $2^b$ | 每块/每行的字节数 | 64 字节（现代 CPU）|
| **m** | 物理地址位数 | 64 位（但通常只用低 48 位）|

**缓存总容量**：$C = S \times E \times B$ 字节（不包括 tag 和 valid 位的开销）

### 5.2 内存地址的划分

给定一个 m 位地址，硬件将其划分为三个字段：

```
地址：|   Tag (标记)   |  Set Index (组索引) |  Block Offset (块内偏移) |
       <── t bits ──>  <──── s bits ────>  <────── b bits ──────>
```

- **Block Offset（块内偏移，b 位）**：指定数据在块内的第几个字节
- **Set Index（组索引，s 位）**：指定数据映射到哪一组
- **Tag（标记，t = m – s – b 位）**：剩余高位，用于区分映射到同一组的不同地址

```
例：m=32 位地址，s=3（S=8组），b=4（B=16字节/块）
地址 0x12345678：
  Block Offset = 0x8 (低 4 位)
  Set Index    = 0x7 (接下来的 3 位)  → 映射到第 7 组
  Tag          = 0x123456 (剩余高 25 位)
```

> **与 Cache Lab 的直接对应**：`set = (address >> b) & (S - 1)`，`tag = address >> (s + b)`。这就是从地址中提取组索引和标记的直接代码实现。

#### 经典例题：从缓存规格推导全部参数

**已知条件**：

- 32 KB 缓存容量（数据部分，不含 tag/valid 开销）
- 8 路组相联（8-way set associative）
- 64 字节每块
- 47 位物理地址

**问题**：推导 B, b, E, S, s, t, C 各参数。

**逐步推导**：

**第 1 步：定 B 和 b**

$$B = 64\ \text{字节（题干直接给出）}$$

$$b = \log_2 B = \log_2 64 = 6$$

b = 6 位 Block Offset。含义：块内 64 字节，需要 6 位来寻址块内的具体字节。

**第 2 步：定 E**

E = 8（题干直接给出："8-way" 即 8 路组相联）。

**第 3 步：由容量公式反推 S**

缓存数据容量公式（**不含 tag/valid 开销**）：

$$C = S \times E \times B$$

$$S = \frac{C}{E \times B} = \frac{32 \times 1024}{8 \times 64} = \frac{32768}{512} = 64$$

S = 64 组。

**第 4 步：由 S 推导 s**

$$s = \log_2 S = \log_2 64 = 6$$

s = 6 位 Set Index。含义：共 64 组，需要 6 位来选中某组。

**第 5 步：由 m, s, b 推导 t**

地址三分法的基本约束：**m = t + s + b**

$$t = m - (s + b) = 47 - (6 + 6) = 35$$

t = 35 位 Tag。含义：除去组索引和块内偏移后，剩余 35 位高位用作标记，区分映射到同一组的不同地址。

**第 6 步（验证）：含 tag/valid 的总物理容量**

$$C_{total} = S \times E \times (1 + t + B \times 8)\ \text{bits}$$

$$= 64 \times 8 \times (1 + 35 + 64 \times 8) = 512 \times (1 + 35 + 512) = 512 \times 548 = 280,576\ \text{bits}$$

$$= \frac{280,576}{8 \times 1024} \approx 34.25\ \text{KB}$$

> 题干给的 32 KB 是纯数据容量，加上 tag（35 位/行）和 valid（1 位/行）的开销后，芯片实际占用约 34.25 KB 的 SRAM。

**汇总表**：

| 参数 | 含义 | 值 | 推导公式 |
|:---:|------|:--:|---------|
| **B** | 每块字节数 | **64** | 题干直接给出 |
| **b** | Block Offset 位宽 | **6** | $\log_2 B$ |
| **E** | 每组的行数（相联度）| **8** | 题干直接给出（8-way）|
| **S** | 组数 | **64** | $C / (E \times B) = 32768 / 512$ |
| **s** | Set Index 位宽 | **6** | $\log_2 S$ |
| **t** | Tag 位宽 | **35** | $m - (s + b)$ |
| **m** | 物理地址位宽 | **47** | 题干直接给出 |
| **C** | 缓存数据容量 | **32 KB** | 题干直接给出 |

**地址三分法图示**：

```
47 位地址：
┌───────────────────────────┬───────────┬───────────┐
│           Tag             │ Set Index │Block Off  │
│         t = 35 位         │ s = 6 位  │ b = 6 位  │
└───────────────────────────┴───────────┴───────────┘
```

> **考场速算口诀**：$C / (E \times B)$ 得 S → $\log_2$ 得 s → $m - s - b$ 得 t。E 和 B 通常是题干直接给的，真正的计算只有这三步。

### 5.3 缓存中的一行（Cache Line）

| 有效位 Valid | 标记 Tag | 数据块 B 字节 |
|:----------:|:-------:|:----------:|
| 1 bit | t bits | B bytes |

- **Valid**：该行是否包含有效数据（初始为 0，加载数据后置 1）
- **Tag**：用于区分映射到同一组的不同地址
- **Block**：从下层存储器复制上来的 B 字节数据

**CPU 怎么判断命中？**
1. 用 Set Index 选中对应的组
2. 在该组中查找 `valid == 1 且 tag 匹配` 的行
3. 找到 → **Hit**，用 Block Offset 取出对应字节
4. 没找到 → **Miss**，去下一层取整个块

### 5.4 三种映射方式

#### 直接映射缓存（Direct-Mapped, E=1）

每组只有一行。给定地址的块只能放在**唯一指定**的那一组中。

```
优点：硬件简单，查找快（只比较一个 tag）
缺点：冲突不命中多——两个热门地址如果映射到同一组，
      会互相挤占（即使缓存其他地方还有大量空闲）
```

**经典冲突场景**：步长为 2 的幂的数组访问：

```c
float a[4096];  // 假设直接映射，S=1024, B=16
// a[0] 和 a[1024] 可能映射到同一组，交替访问 → 每次都 miss！
```

#### 组相联缓存（Set-Associative, E>1）

每组有 E 行。给定地址的块可以放在该组的**任意一行**中。

```
优点：减少冲突不命中，灵活性好
缺点：硬件需要并行比较 E 个 tag → 复杂度和功耗增加
```

现代 CPU 的 L1 数据缓存通常是 8 路组相联，L2/L3 可达 16 路。

#### 全相联缓存（Fully Associative, S=1）

只有一个组（S=1），所有块可以放在**任意位置**。

```
优点：零冲突不命中（只有容量不命中）
缺点：需要并行比较所有行的 tag → 硬件代价极大，只适合小容量缓存
```

TLB（Translation Lookaside Buffer，第 9 章）和虚拟内存的页表本质上就是全相联。

### 5.5 替换策略

当发生 Miss 且目标组**已满**（没有空行）时，需要**驱逐（Evict）**一个旧行：

| 策略 | 规则 | 评价 |
|:---:|------|------|
| **LRU**（Least Recently Used）| 淘汰**最久未被访问**的那一行 | 效果好但硬件实现复杂（需要维护访问时间戳）|
| **FIFO** | 淘汰**最早加载**的那一行 | 简单但可能淘汰热点数据 |
| **随机** | 随机选一行 | 简单，实现成本最低 |

> **Cache Lab 的 LRU 实现**：为每行维护一个 `lru_counter`，每次访问命中时将命中的行 counter 置 0、其余行 counter+1；需要驱逐时找 counter 最大的行。

### 5.6 写策略

#### 写命中（Write Hit）时：

| 策略 | 行为 | 优缺点 |
|:---:|------|--------|
| **写直达**（Write-Through）| 同时更新缓存和下一层 | 简单，但每次写都要等慢速的下一层完成 |
| **写回**（Write-Back）| 只更新缓存，标记为脏（dirty），被驱逐时才写回 | 减少写下一层的次数，但需要 dirty bit，一致性更复杂 |

#### 写不命中（Write Miss）时：

| 策略 | 行为 | 搭配的写命中策略 |
|:---:|------|:--------------:|
| **写分配**（Write-Allocate）| 先把块从下一层读上来，再在缓存中修改 | 通常搭配**写回** |
| **非写分配**（No-Write-Allocate）| 直接写下一层，不加载到缓存 | 通常搭配**写直达** |

> 现代 CPU 的 L1/L2 缓存通常使用 **写回 + 写分配** 策略。

## 六、编写缓存友好的代码（§6.6–§6.7）

### 6.1 基本原则

1. **让常见情况快**：把核心循环的变量和数组控制在工作集大小内
2. **最小化每层的不命中率**：利用好时间局部性和空间局部性
3. **认准访问模式**：
   - **行优先遍历**（空间局部性最优）远优于**列优先遍历**
   - **分块（Tiling）**让一个块被充分利用后再换下一块

### 6.2 矩阵乘法优化 — 从慢到快的四个版本

以下以 $n \times n$ 矩阵乘法 $C = A \times B$ 为例：

#### 版本 1：最朴素的 ijk（按行）

```c
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++) {
        double sum = 0;
        for (int k = 0; k < n; k++)
            sum += A[i][k] * B[k][j];
        C[i][j] = sum;
    }
```

- **A[i][k]**：行优先 → 步长 1 → 空间局部性**较好**
- **B[k][j]**：列优先 → 步长 n → 空间局部性**极差**（每次访问不同的 cache block）
- 对 B 的访问每次都不命中（当 n 较大时）→ **性能最差**

#### 版本 2：ikj（交换循环顺序）

```c
for (int i = 0; i < n; i++)
    for (int k = 0; k < n; k++) {
        double aik = A[i][k];
        for (int j = 0; j < n; j++)
            C[i][j] += aik * B[k][j];
    }
```

- **A[i][k]**：仍然是行优先 → 较好
- **B[k][j]**：内层遍历 j，按行访问 → 步长 1 → **空间局部性优秀**
- **C[i][j]**：内层遍历 j，按行更新 → 步长 1 → 空间局部性优秀
- 比版本 1 快数倍

#### 版本 3：kij（另一种循环顺序）

```c
for (int k = 0; k < n; k++)
    for (int i = 0; i < n; i++) {
        double aik = A[i][k];
        for (int j = 0; j < n; j++)
            C[i][j] += aik * B[k][j];
    }
```

- A 和 B 的访问模式与 ikj 类似，但 C 在内层被反复更新 → 寄存器级复用
- 各版本性能优劣取决于具体的缓存大小和矩阵维度

#### 版本 4：分块 / Tiling

```c
// blockSize 选为能装入 L1 缓存的尺寸
for (int i = 0; i < n; i += B)
    for (int j = 0; j < n; j += B)
        for (int k = 0; k < n; k += B)
            // 内层：在一个 B×B 的 tile 上做矩阵乘法
            for (int i1 = i; i1 < i+B; i1++)
                for (int j1 = j; j1 < j+B; j1++) {
                    double sum = C[i1][j1];
                    for (int k1 = k; k1 < k+B; k1++)
                        sum += A[i1][k1] * B[k1][j1];
                    C[i1][j1] = sum;
                }
```

**为什么分块有效？**

- 不分块时：$B$ 矩阵的每一整行太大，无法全部装进缓存。按列访问 $B$ 会导致每个元素被用一次就被驱逐，下次再用又要从主存加载。
- 分块后：每个 $B \times B$ 的 tile 足够小，可以整个装进 L1 缓存。在 tile 内的所有计算共享同一批缓存中的数据 → **时间局部性最大化**。
- 分块大小 $B$ 的选择：使三个 $B \times B$ 的 tile（A 的、B 的、C 的）能同时装入 L1 缓存。

```
不命中率对比（n=2048, L1=32KB）：
  朴素 ijk：  ~20% miss rate
  ikj：      ~5% miss rate
  ikj+tiling：~0.5% miss rate
```

## 七、重点总结 & 常见考题类型

### 7.1 核心原理一句话

> **存储器层次结构利用程序的时间和空间局部性，让 CPU 觉得既快又大。缓存是硬件自动管理的，对用户程序（除了性能）完全透明。**

### 7.2 必会技能清单

1. **地址分解**：给定 (S, E, B, m)，将一个地址分解为 Tag / Set Index / Block Offset
2. **命中判断**：给定缓存状态和访问序列，判断每次访问是 Hit / Miss / Eviction
3. **计算不命中率**：分析一段 C 代码的缓存行为，解释为什么 miss rate 高或低
4. **局部性分析**：判断一段代码展现哪种局部性（时间/空间），为什么
5. **替换策略**：在组相联缓存中用 LRU 决定驱逐哪一行
6. **写策略**：区分写直达/写回、写分配/非写分配
7. **缓存友好代码**：行优先 > 列优先；分块 tiling 的原理

### 7.3 与第 9 章虚拟内存的直接对应

| 第 6 章概念 | 第 9 章对应 |
|:---------:|:---------:|
| 缓存块（Block）64B | 页面（Page）4KB |
| L1/L2/L3 缓存 SRAM | 物理内存 DRAM |
| 主存 DRAM | 磁盘 SSD/HDD |
| 命中 / 不命中 | 页命中 / 缺页 |
| 写回（Write-Back）| 脏页写回 |
| LRU 替换 | 页面置换算法（LRU/Clock）|
| 全相联（S=1）| 虚拟内存的页表是全相联的 |
| Tag / Set Index / Offset | VPN（虚拟页号）/ VPO（页内偏移）|
| Valid bit | PTE 的有效位 |

> 虚拟内存本质：把主存当作磁盘的"缓存"，用页表替代 tag 数组，用 OS 软件管理替代硬件自动管理——但所有核心概念（命中/不命中/替换/写回/全相联）完全平行。
